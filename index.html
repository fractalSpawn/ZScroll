What is Z-Scrolling?

Traditional web applications mostly scroll along the y-axis, and sometimes the x-axis. With CSS3 transformations and a little jQuery magic, we can have pages that move along the z-axis; creating a 3rd dimension of movement.

Panels can be designed to hold any kind of content and then are either zoomed behind us, or away from us depending on our direction of travel. We can attach this effect to the mousewheel, or to some buttons (for mobile devices).

The effect itself is pretty cool, but depending on how it's implemented, it can lead to an unpleasant user experience. Essentially we are using a single CSS transition on panels, then multiple classes that use CSS transform to scale the panels depending on our direction of travel.

The value that we use for the scaling is our point of contention. Should the value be controlled by the amount of scroll, or just set to predetermined numbers where scrolling just moves from one panel to the next?

When binding the scale value to the scroll amount the panels would get larger and larger and in some browsers the text would flash as giant pixelated letters before being re-rendered as smooth vectors (which is what text is). For images, this could become a nightmare. The effect is a bit more interactive, but not as good of a presentation.

Keep scrolling or scroll back...

To overcome this, I've programmed the approprate scale values into the CSS classes so that a user only needs to scroll a tiny amount to move on the z-axis towards the next panel, or backwards to the previous one. It's slightly less control, but there is no problem of the panels becoming 'ugly' as we move up and down the z-axis.

I've also prevented from over-scrolling by only allowing one transition at a time. Otherwise, the management of css classes becomes problematic and causes the effect to break.

I think you'll agree that while this z-scroll is not quite as fluid as the x or y scrolling we are already accustomed to, the effect is still quite useful when a unique web experience is desired.
